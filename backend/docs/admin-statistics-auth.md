# Спецификация статистики административной панели: Auth

## Назначение

`auth` отвечает только за те метрики раздела статистики, которые принадлежат учётной записи пользователя.

## Что относится к auth

### Общая статистика платформы

- Общее число зарегистрированных пользователей
- Число верифицированных email

### Статистика по конкретному пользователю

- Дата регистрации

## Доступ

Оба endpoint доступны только администратору.

- Аутентификация: `Bearer JWT`
- Если токен отсутствует или невалиден, сервис возвращает `401 Unauthorized`
- Если пользователь не является администратором, сервис возвращает `403 Forbidden`

## Endpoint: общая статистика auth

`GET /auth/admin/statistics/platform`

### Ответ `200 OK`

```json
{
  "users": {
    "totalRegistered": 12450,
    "verifiedEmails": 11803
  }
}
```

### Поля ответа

- `users.totalRegistered` — общее количество зарегистрированных аккаунтов
- `users.verifiedEmails` — количество аккаунтов с подтверждённым email

### Ошибки

- `401 Unauthorized`
- `403 Forbidden`
- `500 Internal Server Error`

## Endpoint: статистика auth по пользователю

`GET /auth/admin/users/{id}/statistics`

### Path parameters

- `id` — UUID пользователя

### Ответ `200 OK`

```json
{
  "userId": "123e4567-e89b-12d3-a456-426614174000",
  "registeredAt": "2026-03-04T12:00:00Z",
  "emailVerified": true
}
```

### Поля ответа

- `userId` — UUID пользователя
- `registeredAt` — дата и время регистрации
- `emailVerified` — подтверждён ли email пользователя

`emailVerified` добавлен в ответ как служебное поле auth-контекста. Оно не обязательно для пользовательского блока статистики, но естественно относится к `auth` и может быть полезно в административной панели.

### Ошибки

- `401 Unauthorized`
- `403 Forbidden`
- `404 Not Found` — пользователь не найден
- `500 Internal Server Error`

## Замечания по границам ответственности

- Метрика `Число заблокированных пользователей` не описана
- Репутация, подписки и история репутационных событий не относятся к `auth`; их должен отдавать `users`
- Объявления, сделки, жалобы и отзывы не относятся к `auth`; их должен отдавать `deals`
- Число чатов относится к `chats`
