# Barter Port

Платформа для бартерного обмена товарами и услугами без денежных расчётов.

---

## Стек

| Слой | Технологии |
|------|-----------|
| Backend | Go 1.26, Chi v5, pgx v5, golang-migrate, gRPC, Kafka |
| Frontend | React 19, TypeScript, Vite, MUI, RTK Query, Zod |
| Базы данных | PostgreSQL (по одной на каждый сервис) |
| Хранилище файлов | SeaweedFS (S3-совместимый API) |
| Прокси | Caddy |
| Инфраструктура | Docker Compose |

---

## Архитектура

Четыре независимых Go-сервиса. Связь между сервисами — gRPC. Асинхронные события (репутация, штрафы) — через Kafka. Фронтенд общается с сервисами через Caddy-прокси.

```
Browser → Caddy (80)
              ├── /api/auth/   → Auth    (8081)
              ├── /api/deals/  → Deals   (8080)
              ├── /api/users/  → Users   (8082)
              ├── /api/chats/  → Chats   (8083)
              └── /s3          → SeaweedFS (8333)

Auth ←gRPC→ Deals, Users, Chats
Users ←gRPC→ Deals, Chats
Deals → Kafka → Users (репутационные события)
```

### Сервисы

| Сервис | HTTP | gRPC | Ответственность |
|--------|------|------|----------------|
| **Auth** | 8081 | 50051 | Регистрация, вход, JWT, email-верификация, сброс пароля, управление администраторами |
| **Deals** | 8080 | 50054 | Объявления, сделки, черновики, жалобы, отзывы, статистика, геолокация |
| **Users** | 8082 | 50052 | Профили, репутация, подписки, чёрный список |
| **Chats** | 8083 | 50053 | Чаты, сообщения |

## Быстрый старт

### Требования

- Docker и Docker Compose
- Go 1.26+ (только для локальной разработки)
- Node.js 18+ (только для разработки фронтенда)

### 1. Переменные окружения

Скопируйте и заполните `.env`:

```bash
cp .env.example .env
```

Обязательные переменные:

```env
DB_PASSWORD=postgres
JWT_ACCESS_SECRET=<случайная строка>
JWT_REFRESH_SECRET=<случайная строка>
S3_ACCESS_KEY_ID=barter-port-s3
S3_SECRET_ACCESS_KEY=barter-port-s3-secret
S3_REGION=us-east-1
```

### 2. Запуск инфраструктуры и приложения

```bash
# Поднять инфраструктуру (Postgres, Kafka, SeaweedFS, SMTP, Swagger)
make infra-up

# Применить миграции
make migrate-up

# Поднять сервисы приложения
make app-up
```

Или одной командой с тестовыми данными:

```bash
make reseed-demo
```

Приложение доступно на `http://localhost`.

### 3. Локальная разработка без Docker

```bash
# Запустить все 4 сервиса локально (нужна запущенная инфраструктура)
make local-up
```

Фронтенд:

```bash
cd frontend
npm install
npm run dev   # http://localhost:5173
```

## Makefile — основные команды

```bash
make infra-up            # Запустить инфраструктуру
make infra-down          # Остановить инфраструктуру
make infra-down-v        # Остановить и удалить тома

make app-up              # Запустить сервисы приложения
make app-down            # Остановить сервисы
make app-down-v          # Остановить и удалить тома

make migrate-up          # Применить все миграции
make migrate-reset-all   # Откатить и применить все миграции заново

make seed-demo           # Загрузить тестовые данные
make reseed-demo         # infra-up + migrate-up + seed-demo

make local-up            # Запустить сервисы локально (без Docker)

make create-migration db=deals name=add_field   # Создать новую миграцию
make grpcgen             # Перегенерировать gRPC-код из proto-файлов
```

## Реализованные функции

### Пользователи
- Регистрация с подтверждением email
- Вход, выход, обновление JWT
- Сброс пароля по email
- Профиль: имя, bio, аватар (хранится в S3), телефон, текущее местоположение
- Подписки и подписчики
- Чёрный список (скрытие постов, ограничение сообщений и заявок)
- История репутационных событий

### Объявления
- Создание и редактирование объявлений (название, описание, тип, теги, фото)
- Категории: `good` / `service`, направление: `give` / `take`
- Фото (до 10 штук, до 5 МБ каждое, хранятся в S3)
- Геолокация: точка на Яндекс Картах; расстояние от пользователя до оффера считается по формуле Хаверсина
- Группы объявлений
- Избранное
- Счётчик просмотров
- Жалобы с модерацией администратором

### Сделки
- Инициация сделки по объявлению
- Черновики условий
- Жизненный цикл: переговоры → завершение / провал
- Фиксация провала, назначение виновного
- Чат участников сделки

### Репутация
- +5 баллов за завершённую сделку
- +2 балла за оставленный отзыв
- Штрафные баллы за жалобы и провалы сделок
- События передаются через Kafka (outbox-паттерн)

### Отзывы
- Отзывы на офферы и их авторов
- Средний рейтинг

### Администрирование
- Модерация жалоб на объявления
- Управление тегами
- Видимость чатов

### API-документация
- Swagger UI доступен на `http://localhost/docs`

## Переменные окружения для фронтенда

Создайте `frontend/.env.local`:

```env
VITE_API_URL=http://localhost
VITE_YANDEX_MAPS_API_KEY=<ваш ключ Яндекс Карт>
```

Ключ Яндекс Карт нужен для карты выбора местоположения в офферах и профиле. Без ключа карта не загрузится, но остальной функционал работает.
