# Barter Port

Платформа для бартерного обмена товарами и услугами без денежных расчётов.

---

## Стек

| Слой | Технологии |
|------|-----------|
| Backend | Go 1.26, Chi v5, pgx v5, golang-migrate, gRPC, Kafka |
| Frontend | React 19, TypeScript, Vite, MUI, RTK Query, Zod |
| Базы данных | PostgreSQL (по одной на каждый сервис) |
| Хранилище файлов | SeaweedFS (S3-совместимый API) |
| Прокси | Caddy |
| Инфраструктура | Docker Compose |

---

## Архитектура

Четыре независимых Go-сервиса. Связь между сервисами — gRPC. Асинхронные события (репутация, штрафы) — через Kafka. Фронтенд общается с сервисами через Caddy-прокси.

```
Browser → Caddy (80)
              ├── /api/auth/   → Auth    (8081)
              ├── /api/deals/  → Deals   (8080)
              ├── /api/users/  → Users   (8082)
              ├── /api/chats/  → Chats   (8083)
              └── /s3          → SeaweedFS (8333)

Auth ←gRPC→ Deals, Users, Chats
Users ←gRPC→ Deals, Chats
Deals → Kafka → Users (репутационные события)
```

### Сервисы

| Сервис | HTTP | gRPC | Ответственность |
|--------|------|------|----------------|
| **Auth** | 8081 | 50051 | Регистрация, вход, JWT, email-верификация, сброс пароля, управление администраторами |
| **Deals** | 8080 | 50054 | Объявления, сделки, черновики, жалобы, отзывы, статистика, геолокация |
| **Users** | 8082 | 50052 | Профили, репутация, подписки, чёрный список |
| **Chats** | 8083 | 50053 | Чаты, сообщения |

## Быстрый старт

### Требования

- Docker и Docker Compose
- Go 1.26+ (только для локальной разработки)
- Node.js 18+ (только для разработки фронтенда)

### 1. Переменные окружения

Скопируйте и заполните `.env`:

```bash
cp .env.example .env
```

Обязательные переменные:

```env
DB_PASSWORD=postgres
JWT_ACCESS_SECRET=<случайная строка>
JWT_REFRESH_SECRET=<случайная строка>
S3_ACCESS_KEY_ID=barter-port-s3
S3_SECRET_ACCESS_KEY=barter-port-s3-secret
S3_REGION=us-east-1
```

### 2. Запуск инфраструктуры и приложения

```bash
# Поднять инфраструктуру (Postgres, Kafka, SeaweedFS, SMTP, Swagger)
make infra-up

# Применить миграции
make migrate-up

# Поднять сервисы приложения
make app-up
```

Или одной командой с тестовыми данными:

```bash
make reseed-demo
```

Приложение доступно на `http://localhost`.

### 3. Локальная разработка без Docker

```bash
# Запустить все 4 сервиса локально (нужна запущенная инфраструктура)
make local-up
```

Фронтенд:

```bash
cd frontend
npm install
npm run dev   # http://localhost:5173
```


## Реализованные функции

### Пользователи
- Регистрация с подтверждением email
- Вход, выход, обновление JWT
- Сброс пароля по email
- Профиль: имя, bio, аватар (хранится в S3), телефон, текущее местоположение
- Подписки и подписчики
- Чёрный список (скрытие постов, ограничение сообщений и заявок)
- История репутационных событий

### Объявления
- Создание и редактирование объявлений (название, описание, тип, теги, фото)
- Категории: `good` / `service`, направление: `give` / `take`
- Фото (до 10 штук, до 5 МБ каждое, хранятся в S3)
- Геолокация: точка на Яндекс Картах; расстояние от пользователя до оффера считается по формуле Хаверсина
- Группы объявлений
- Избранное
- Счётчик просмотров
- Жалобы с модерацией администратором

### Сделки
- Инициация сделки по объявлению
- Черновики условий
- Жизненный цикл: переговоры → завершение / провал
- Фиксация провала, назначение виновного
- Чат участников сделки

### Репутация
- +5 баллов за завершённую сделку
- +2 балла за оставленный отзыв
- Штрафные баллы за жалобы и провалы сделок
- События передаются через Kafka (outbox-паттерн)

### Отзывы
- Отзывы на офферы и их авторов
- Средний рейтинг

### Администрирование
- Модерация жалоб на объявления
- Управление тегами
- Видимость чатов

### API-документация
- Swagger UI доступен на `http://localhost/docs`

Создайте `frontend/.env.local`:

```env
VITE_API_URL=http://localhost
VITE_YANDEX_MAPS_API_KEY=<ваш ключ Яндекс Карт>
```

Ключ Яндекс Карт нужен для карты выбора местоположения в офферах и профиле. Без ключа карта не загрузится, но остальной функционал работает.

## Функциональные требования

### Действия с аккаунтом

1. Зарегистрироваться по email с подтверждением адреса через письмо.
2. Войти в аккаунт по email и паролю; выйти из аккаунта.
3. Восстановить доступ к аккаунту по email (ссылка для сброса пароля).
4. Сменить имя, пароль, аватар, bio, контактный телефон и текущее местоположение в профиле.
5. Посмотреть историю репутационных событий (начисления и штрафы с источниками).
6. Посмотреть свою статистику: количество завершённых, активных и провальных сделок, данные по отзывам.
7. Посмотреть избранные объявления.
8. Посмотреть активные сделки и черновики.

### Поиск объявлений

1. Просматривать каталог объявлений, отсортированный по новизне или популярности.
2. Искать объявления рядом (сортировка по расстоянию на основе геолокации пользователя).
3. Фильтровать объявления по тегам (категориям).
4. Просматривать объявления пользователей, на которых оформлена подписка, в отдельной вкладке.
5. Откликнуться на объявление или сценарий обмена, тем самым создав черновик сделки.
6. Пожаловаться на объявление.
7. Добавить объявление в избранное.

### Выставление объявления

1. Опубликовать объявление с желаемым или предлагаемым товаром либо услугой (направление «ищу» / «отдаю», тип «товар» / «услуга»).
2. Добавить к объявлению название, описание, теги, до 10 фотографий и точку на Яндекс Картах.
3. Объединить несколько собственных объявлений в сценарий обмена с блоками и логикой AND/OR между ними.
4. Редактировать и удалять собственное объявление; скрывать его от других пользователей.

### Активные сделки

1. Присоединиться к открытой сделке (в статусе «Поиск участников»); покинуть сделку.
2. Принять или отклонить заявку другого пользователя на вступление в сделку.
3. Удалить участника из сделки путём голосования остальных участников.
4. Добавлять, редактировать и просматривать позиции сделки (товар/услуга с фотографиями, роли провайдера и получателя).
5. Вести переговоры в чате сделки.
6. Подтвердить условия сделки; голосовать за переход статуса.
7. Зафиксировать провал сделки с указанием виновного; администратор рассматривает спор и выносит решение.

### Отзывы

1. Оставить отзыв на позицию сделки и её автора после успешного завершения.
2. Просматривать, редактировать и удалять собственные отзывы.
3. Просматривать средний рейтинг и все отзывы об авторе объявления на его странице.

### Взаимодействие пользователей

1. Подписаться на другого пользователя; отписаться; просматривать список своих подписок и подписчиков.
2. При взаимной подписке — создать личный чат и вести переписку без ограничений.
3. Общаться с участниками сделки в чате сделки.
4. Добавить пользователя в чёрный список, скрыв от него свои объявления и ограничив возможность отклика и переписки.

### Геймификация

1. Получать репутационные очки: +5 за завершённую сделку, +2 за оставленный отзыв.
2. Получать штрафные баллы за принятые жалобы на объявления и за провалы сделок.

### Функции администратора

1. Просматривать очередь жалоб на объявления и принимать решение (принять / отклонить) с комментарием.
2. Просматривать спорные случаи провала сделок, голоса участников и принимать решение о назначении штрафа.
3. Управлять тегами: просматривать и удалять.

## Статистика в административной панели

Раздел статистики доступен администратору и содержит две части: общие метрики платформы и метрики по конкретному пользователю.

### Общая статистика платформы

#### Пользователи
- Общее число зарегистрированных пользователей
- Число верифицированных email
- Число заблокированных пользователей
- Число пользователей с перманентной блокировкой
- Распределение пользователей по уровням репутации

#### Объявления
- Общее число объявлений
- Число активных объявлений (на будущее)
- Число скрытых объявлений: по цензуре либо самим пользователем
- Распределение по типу: товар / услуга
- Распределение по направлению: «ищу» / «отдаю»
- Топ тегов/категорий по числу объявлений
- Среднее число объявлений на одного пользователя
- Число черновиков
- Общее число просмотров объявлений за период
- Топ объявлений по числу добавлений в избранное
- Средний рейтинг объявлений по платформе

#### Сделки
- Общее число сделок
- Распределение сделок по статусам: поиск участников / готова к подтверждению / подтверждена / успех / провал / отмена
- Процент успешных сделок от общего числа завершённых (conversion rate)
- Среднее число участников в сделке
- Доля многосторонних сделок (более двух участников)
- Среднее время жизни сделки от создания до завершения (незавершенные не считаются)

#### Жалобы и модерация
- Общее число жалоб на объявления
- Число необработанных жалоб (текущая очередь)
- Число заблокированных объявлений
- Число вынесенных решений администратора по спорам
- Топ пользователей по числу полученных жалоб

#### Репутация и геймификация
- Средняя репутация по всем пользователям
- Медианная репутация
- Топ-N пользователей по репутации

#### Отзывы
- Общее число отзывов на платформе
- Средний рейтинг по всей платформе
- Распределение оценок (1–5 звёзд)

#### Чаты и сообщения
- Общее число чатов
- Общее число сообщений за период

---

### Статистика по конкретному пользователю

- Число завершённых сделок
- Число активных сделок
- Число провальных сделок с указанием роли (виновный / пострадавший)
- Число отменённых сделок
- Число опубликованных объявлений
- Суммарное число просмотров объявлений пользователя
- Текущий репутационный балл и история начислений/штрафов
- Число полученных отзывов и средний рейтинг
- Число оставленных отзывов
- Число жалоб, полученных пользователем (принятых / отклонённых)
- Число жалоб, отправленных пользователем
- Число подписчиков и подписок
- Дата регистрации
